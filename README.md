# Bellabeat Case Study (v2)

This project analyzes smart device usage data to help Bellabeat, a high-tech wellness company, understand how consumers use their products. The goal is to identify trends and provide insights that can guide marketing strategy.

---

## 📊 Project Overview

This case study follows the full data analytics process:

1. **Ask** – Define the business task and key questions  
2. **Prepare** – Collect and organize the Fitbit dataset  
3. **Process** – Clean and transform the data  
4. **Analyze** – Explore trends in activity, sleep, and heart rate  
5. **Share** – Build visuals and communicate insights  
6. **Act** – Recommend data-driven marketing strategies  

---

## 🧹 Data Sources

- Fitbit Fitness Tracker Data (public dataset)
- Daily, hourly, and intensity-level activity data
- Sleep and weight logs (where available)

---

## 🛠️ Tools Used

- **R** (tidyverse, lubridate, ggplot2)
- **Excel** (initial exploration)
- **GitHub** (version control)
- **PowerPoint** (final presentation)

## 📈 Key Insights

- Users with higher activity levels tend to have more consistent sleep patterns  
- Most users are active during morning and early evening hours  
- Smart device usage suggests opportunities for targeted wellness reminders  
- There is strong potential for personalized marketing based on activity trends  

---

## 📝 Recommendations

- Promote features that encourage daily activity streaks  
- Develop sleep‑focused wellness content  
- Use push notifications during peak activity windows  
- Build campaigns around personalized health insights
